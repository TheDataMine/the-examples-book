Now let’s discuss constraints.

In SciPy, an inequality constraint is a function defined so that:

[source,python]
----
constraint_fun() >= 0
----

For example:
- To require `x >= 0`:  
  `lambda x: x[0]`
- To require `x <= 5`:  
  `lambda x: 5 - x[0]`
- To require `x >= 10`:
  `lambda x: x[0] - 10`

On the contraty, an equality constraint is a function defined so that:

[source,python]
----
constraint_fun() == 0
----

For example:
- To require `x == 10`:
  `lambda x: x[0] - 10`
- In a multivariable problem, to require `x + y == 10`:
  `lambda x, y: x[0] + y[0] - 10`

You can also have more complex constraints, known as nonlinear constraints. These are constraints that are not linear in the variables. For example:
- require `x` is an even number:
  `lambda x: x[0] % 2`
- require `x` is a prime number:
  `lambda x: is_prime(x[0])`
- require `last digit in x is a 3 or a 7`:
  `lambda x: x[0] % 10 in [3, 7]`
