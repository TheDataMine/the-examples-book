=== Question 1: Understanding LSB & Encoding (2 points)

==== What is LSB Steganography?

Steganography is the art and science of hiding information within otherwise non-secret data. You can accomplish this by subtly modifying pixel values. The Least Significant Bit (LSB) method is one of the simplest and most intuitive approaches.

**Key Concept**: Each pixel in a grayscale image has a value from 0-255 (8 bits). The least significant bit (rightmost bit) contributes the least to the visual appearance. Changing it from 0 to 1 (or vice versa) changes the pixel value by at most 1, which is imperceptible to the human eye.

For example:

- Pixel value 150 in binary: `10010110` (LSB = 0)
- Change LSB to 1: `10010111` = 151

This is a subtle change, but it is enough to hide a message while staying hidden.

==== Step 1: Converting Messages to Binary

Before we can hide a message, we need to convert it to binary since images are just composed of a bunch of pixels, which are in turn composed of bits. Each character of our message can be decomposed into 8 bits. We will also need a special "delimiter" to mark where the message ends so we do not accidentally decode a message that is not intended for us.

==== Step 2: Understanding How to Modify LSBs

Now that, we have our message in binary, we need to modify the LSBs of each pixel in our image to hide our message. To encode a bit into a pixel's LSB, we need to:

1. Extract the current LSB, +

==== Step 2: Adaptive LSB Based on Variance

As we saw with the MSE and PSNR values using n-LSB encoding, the difference is much more noticable. You may have even noticed something in the image itself if you used a large enough value for `n` with a long enough message. 

Another way to approach encoding more information in an image is using some form of dynamic analysis to decide where you can reasonably encode more information. This stems from the fact that some images are "messier" by nature - think of a picture of Times Square at night versus a picture of a simple grass field and with a clear blue sky. The picture of Time Square has more going on all the different lights, advertisements, people, cars, etc compared to the simplistic block of green and block of blue.

With this in mind, we can look at implementing adaptive encoding based on the pixels _regional variance_ to ensure we only modify pixels in a higher variance (or already otherwise noisy) area. This is a good way to ensure we are not modifying the image in a way that is easily detectable.

**Key Concept**: Calculate local variance for each pixel. High variance = textured area (good for hiding). Low variance = smooth area (bad for hiding).

They way we are going to approach this is to first create a variance map in a function called `calculate_local_variance(image, window_size=3)`. We need to create a sliding window or a kernel that can give us the average pixel value in the area surrounding the pixel. === Question 4: Frequency Domain Methods - Discrete Cosine Transform (2 points)

==== Understanding Frequency Domain Transformations

So far we've worked in the **spatial domain** (directly modifying pixel values). Now we'll explore the something a little more interesting: the **frequency domain**!

In this approach we look to decompose the image into its frequency components, hide information in the transform coefficients, then transform it back to the spatial domain ideally achieving a more naturally distributed message across the image making it more difficult to detect.

**Why frequency domain?** 

- More robust to compression (like JPEG)
- Can hide in less perceptually important frequencies
- Harder to detect with simple statistical methods (which we will explore in the next question)

We will explore two common frequency domain transformations: the Discrete Cosine Transform (DCT) and the Discrete Wavelet Transform (DWT) as means of hiding information in the frequency domain.

==== Part A: Understanding DCT (Discrete Cosine Transform)

**What is DCT?** DCT converts an image from spatial to frequency representation. It's the same transform used in JPEG compression!

**Key Concept**: 

- Low frequencies = smooth changes, important for visual quality
- High frequencies = sharp edges/details, less important
- Mid frequencies = good place to hide data (not too important, not too noisy)

DCT works on 8*8 blocks:

1. Divide image into 8*8 blocks
2. Apply DCT to each block → get 64 coefficients (now in the frequency domain)
3. Modify selected coefficients (e.g., positions (3,4), (4,3), (4,4))
4. Apply inverse DCT → reconstruct image (now back in the spatial domain)

Looking at this example, we see the image is broken up into blocks and then each block is transformed into its frequency domain representation in the image on the right.

from scipy.fftpack import dct, idct

def encode_dct(image_array, message, block_size=8):
    """
    Encode message using DCT coefficients.
    
    We'll modify mid-frequency coefficients in each 8*8 block.
    """
    # Convert message to binary
    message_binary = ''.join(format(ord(char), '08b') for char in message)
    message_binary += '1111111111111110'
    
    # Ensure image dimensions are multiples of block_size - pad if needed
    h, w = image_array.shape
    h_padded = ((h + block_size - 1) // block_size) * block_size
    w_padded = ((w + block_size - 1) // block_size) * block_size
    
    padded_image = np.zeros((h_padded, w_padded), dtype=float)
    padded_image[:h, :w] = image_array.astype(float)
    
    stego_image = padded_image.copy()
    bit_idx = 0
    
    # Process each 8*8 block
    for i in range(0, h_padded, block_size):
        for j in range(0, w_padded, block_size):
            if bit_idx >= len(message_binary):
                break
            
            # Extract block
            block = padded_image[i:i+block_size, j:j+block_size]
            
            # Apply 2D DCT (apply along rows, then columns) - this is the frequency domain transform
            dct_block = dct(dct(block, axis=0, norm='ortho'), axis=1, norm='ortho')

==== Part B: Decoding the DCT Message

Lets now implement the DCT decoding function! Similar to the LSB encoding function, it is simply the reverse logic of the encoding function:

1. Apply forward DCT to each block to get frequency domain coefficients
2. Extract the modified coefficient at (3,4) by rounding and getting the LSB
3. Convert the bits back to binary string
4. Convert the binary back to characters (stop when delimiter is found)

[source,python]
----
def decode_dct(stego_image, block_size=8, threshold=5.0, delimiter='1111111111111110'):
    """
    Decode message using DCT coefficients.
    """
    # Step 0: Pad the image to be multiples of block_size
    h, w = stego_image.shape
    h_padded = ((h + block_size - 1) // block_size) * block_size
    w_padded = ((w + block_size - 1) // block_size) * block_size

    padded_image = np.zeros((h_padded, w_padded), dtype=float)
    padded_image[:h, :w] = stego_image.astype(float)
    
    message_bits = []

    # Step 1: Apply forward DCT to each block to get frequency domain coefficients
    for i in range(0, h_padded, block_size):
        for j in range(0, w_padded, block_size):
            # Extract block and apply forward DCT:
            block = padded_image[i:i+block_size, j:j+block_size]
            dct_block = dct(dct(block, axis=0, norm='ortho'), axis=1, norm='ortho')
            
            # Step 2: Extract the modified coefficient at (3,4)
            if abs(dct_block[3, 4]) >= threshold:
                # Round the coefficient and extract the LSB:
                coef_rounded = round(dct_block[3, 4])
                bit = str(coef_rounded & 1) # extract the LSB
                message_bits.append(bit)
                
                # Step 3: Check for delimiter to stop decoding early
                found, message_binary = check_delimiter(''.join(message_bits), delimiter)
                if found:
                    # Step 4: Convert the binary back to characters
                    message = ''.join(chr(int(message_binary[i:i+8], 2)) for i in range(0, len(message_binary), 8))
                    return message
    return None

def check_delimiter(message_binary_str, delimiter):
    """
    Check if delimiter is found at the end of the binary string.
    Returns: (found, message_binary_without_delimiter)
    """
    if len(message_binary_str) >= len(delimiter):
        if message_binary_str[-len(delimiter):] == delimiter:
            return True, message_binary_str[:-len(delimiter)]
    return False, None

=== Question 5: Detection Methods - Histogram Analysis (2 points)

==== Detecting Hidden Messages

So far we have talked about different ways to encode messages into images, how to decode them and mentioned how certain methods are more discrete than others. But how can we actually detect if a message has been hidden in an image? This is where steganalysis comes in! 

Steganalysis is the detection of hidden messages. Even though the goals of the methods we have discussed so far is to remain visually imperceptible, LSB in particular, leaves statistical traces that can be detected!

==== Part A: Histogram Analysis

When we perform LSB steganography, we're modifying the least significant bit of pixel values. This creates a specific statistical pattern that can be detected through histogram analysis which can be referred to as the _pairing effect_.

But why do these pairs form? 

Consider what happens when we modify the LSB of a pixel:

- A pixel with value `150` (binary: `10010110`) has LSB = `0`
- If we need to encode a `1`, we change it to `151` (binary: `10010111`)
- Similarly, a pixel with value `151` (LSB = `1`) can be changed to `150` if we need to encode a `0`

The key insight is that when we randomly modify LSBs to encode a message:

- Values `150` and `151` can be converted into each other
- Over many modifications, these pairs should appear with _similar frequencies_

However, in a natural image, there's no real reason why `150` and `151` should have similar counts which is why we can use a histogram approach to reveal this pattern!

What to look for:

1. Pairing Pattern: Adjacent even-odd pairs (0-1, 2-3, 4-5, ..., 254-255) should have similar frequencies
2. Statistical Moments: 
   - _Skewness_: Measures asymmetry. LSB steganography can make distributions more symmetric
   - _Kurtosis_: Measures "tailedness". Steganography can flatten the distribution
   - _Variance_: May change slightly as pixel values are modified

When plotting histograms, you might notice that pairs of values appear more balanced in stego-images compared to originals, especially in regions where many pixels were modified. This can indicate that someone tampered with the image to hide some information in it!

Looking at a more concrete theoretical example:

Imagine an original image has these pixel value counts:

- Value 150: 1200 pixels
- Value 151: 800 pixels
- Total in pair: 2000 pixels

After encoding a random message using LSB steganography (modifying ~50% of pixels with values 150 or 151):

- Some 150s become 151s (when we need to encode a 1)
- Some 151s become 150s (when we need to encode a 0)
- Over many random modifications, these tend to balance out
- Result might be: 150: ~1000 pixels, 151: ~1000 pixels

The histogram would show the pair becoming more balanced, which is suspicious for a natural image!

So now let's analyze histograms on our images to detect these patterns.

from scipy.stats import skew, kurtosis

def analyze_histogram(image_array):
    """
    Analyze histogram statistics of an image.
    
    Returns statistical measures that might reveal steganography.
    """
    flattened = image_array.flatten()
    
    # Calculate histogram
    hist, bins = np.histogram(flattened, bins=256, range=(0, 256))
    
    # Statistical moments
    stats = {
        'mean': np.mean(flattened),
        'variance': np.var(flattened),
        'skewness': skew(flattened),
        'kurtosis': kurtosis(flattened)
    }
    
    return hist, bins, stats

# Compare histograms
def compare_histograms(original, stego):
    """Compare histograms of original and stego-images."""
    hist_orig, bins, stats_orig = analyze_histogram(original)
    hist_stego, _, stats_stego = analyze_histogram(stego)
    
    # Plot side-by-side
    fig, axes = plt.subplots(1, 2, figsize=(14, 5))
    
    axes[0].bar(bins[:-1], hist_orig, width=1, alpha=0.7, label='Original')
    axes[0].set_title('Original Image Histogram')
    axes[0].set_xlabel('Pixel Value')
    axes[0].set_ylabel('Frequency')
    axes[0].legend()
    
    axes[1].bar(bins[:-1], hist_stego, width=1, alpha=0.7, label='Stego', color='orange')
    axes[1].set_title('Stego-Image Histogram')
    axes[1].set_xlabel('Pixel Value')
    axes[1].set_ylabel('Frequency')
    axes[1].legend()
    
    plt.tight_layout()
    plt.show()
    
    # Print statistics
    print("Original Image Statistics:")
    for key, value in stats_orig.items():
        print(f"  {key}: {value:.4f}")
    
    print("\nStego-Image Statistics:")
    for key, value in stats_stego.items():
        print(f"  {key}: {value:.4f}")
    
    return stats_orig, stats_stego

==== Limitations and Considerations

**Histogram Analysis Limitations:**

1. **Visual Inspection is Subjective**: While statistical measures help, visual histogram comparison can be misleading. Natural images can sometimes have balanced pairs by coincidence.

2. **Sensitive to Image Content**: Images with specific characteristics (e.g., very uniform regions, gradients) may naturally show pairing patterns that look like steganography.

3. **Not Quantitative**: Histogram analysis gives you a "feeling" but doesn't provide a clear yes/no answer about steganography. It's more of a qualitative assessment.

4. **Works Best with High Embedding Rates**: If only a small percentage of pixels are modified, the histogram changes may be too subtle to detect visually.

5. **Only Detects LSB Replacement**: 
   - This method specifically targets simple LSB replacement
   - More sophisticated methods (LSB matching, adaptive encoding) may evade detection
   - DCT and DWT steganography won't be detected by this method

6. **Image Processing Effects**:
   - Compression, filtering, or other processing can destroy the pairing pattern
   - The analysis assumes the image hasn't been significantly altered after steganography

**Best Practices:**

- **Test Multiple Images**: Don't rely on a single test; test multiple images to build confidence
- **Compare Statistics**: Look at the statistical measures (mean, variance, skewness, kurtosis) between original and stego images
- **Understand Context**: Consider the image type, size, and any processing it may have undergone
- **Visual Inspection**: Look for pairing patterns in the histogram, especially for even-odd value pairs

**Why This Method Matters:**

Even with limitations, histogram analysis is important because:
1. It reveals that "invisible" steganography isn't truly invisible
2. It demonstrates the importance of statistical security in steganography
3. It motivates the development of more sophisticated steganographic methods (like adaptive encoding, DCT, DWT) that are harder to detect
